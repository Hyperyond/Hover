# Introduction

Hover is a **VS Code extension** (`hover-dev`) that puts an AI chat in your editor. You type a natural-language instruction like *"test the login flow"*. Hover spawns the coding-agent CLI you already run (Claude Code / OpenAI Codex), which drives your *actual* dev Chrome over CDP, narrates each step, and crystallizes the verified session into a standard Playwright spec under `__vibe_tests__/`.

The differentiator vs. Stagehand / Midscene / Playwright codegen is the **AI exploration → deterministic script** workflow: AI authors the test, but the saved artifact is plain `@playwright/test` code that runs in CI without an agent in the loop.

Nothing changes in your app, your bundler config, or your dependencies — the extension drives your existing dev server over CDP regardless of framework (Vite, Next.js, Nuxt, Astro, webpack).

Hover also covers **application security testing** in the same chat — flip into 🟠 **[API testing](/docs/features/api-test/)** mode (IDOR / broken authorization / parameter-tampering, via a local HTTPS MITM) or 🔴 **Pentest** mode (offensive SQLi / XSS / SSTI / SSRF / IDOR against your **own** dev app). Confirmed findings crystallize into `.api-test.spec.ts` CI gates or a findings report.

Building Hover itself? See **[Development](/docs/development/)** for the monorepo workflow.



## At a glance

- **No API key required.** Hover spawns the coding-agent CLI you already have on PATH (`claude`, `codex`, …) and reuses the subscription you already pay for — or [set a model API key](/docs/get-started/api-key/) if you'd rather.
- **Nothing to wire into your app.** The extension drives your existing dev server over CDP — no plugin, no config edit, no new dependency. Works with any framework.
- **Local-first.** The engine (`@hover-dev/core`) binds to `127.0.0.1` only, has no telemetry, and no upload path. Cookies and localStorage never transit it.
- **One artefact, plain Playwright.** Save the verified session as a `@playwright/test` spec that checks into git and runs in CI with zero AI, no tokens, no key.

<Callout type="note" title="No bundler plugin to install">

Hover's original surface was an in-page widget injected via bundler plugins. That path has been removed — the VS Code extension drives any dev server over CDP with nothing to add to your project. `@hover-dev/core` lives on as the extension's engine.

</Callout>
