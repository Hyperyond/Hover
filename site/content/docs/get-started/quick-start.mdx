# Quick start

Install the Hover VS Code extension, install one coding-agent CLI, and send your first prompt. No changes to your app, bundler config, or dependencies.

## Prerequisites

- **VS Code** (or a VS Code-compatible editor).
- A dev server you already run for your project — any framework Hover can reach over CDP (Vite, Next.js, Nuxt, Astro, webpack).
- Any supported coding-agent CLI on PATH — `claude`, `codex`, `cursor-agent`, `aider`, `gemini-cli`, or `qwen-code`. Hover spawns whichever you already have. No new API keys.

<Callout type="tip" title="Don't have an agent CLI yet?">

- **Claude Code** — `npm install -g @anthropic-ai/claude-code`, then `claude login`. Uses the Claude Pro / Max subscription you might already pay for.
- **OpenAI Codex** — `npm install -g @openai/codex`, then `codex login`.

Either works. You can switch from the chat header or Settings any time.

</Callout>

## Install the extension

Open the Extensions view (`⇧⌘X` / `Ctrl+Shift+X`), search **"Hover — AI E2E Testing & Security"**, and click Install — or grab it from the **[VS Code Marketplace](https://marketplace.visualstudio.com/items?itemName=hyperyond.hover-dev)**.

The extension ships the engine inside it. There's nothing to install into your project: no plugin, no bundler-config edit, no new dependency.

## Start your dev server

Use whatever command you already use:

```bash
pnpm dev          # or `npm run dev`, `yarn dev`, `bun dev`
```

Hover drives your existing dev server over CDP — it doesn't replace, wrap, or restart it.

<Callout type="tip" title="First-run debug Chrome">

On your first run, Hover prompts you to launch an **isolated debug Chrome** on port 9222 (separate from your everyday browser — a clean profile under `<tmpdir>/hover-chrome`). Accept the prompt and a debug Chrome opens, navigated to your dev URL. Subsequent runs reuse it. Hover never touches your normal Chrome session.

</Callout>

## Send your first prompt

Open the **Hover** chat from the Activity Bar. Type, or hold the 🎙 button and speak:

```
log in, then add a todo named "verify hover"
```

Press `↵` or **Send**. The agent drives the debug Chrome, narrates each tool call, and renders a Result + Findings card when done.

<Callout type="tip" title="Logging in with @account">

Define test accounts per environment once in the **Environments** view, then just mention `@account` in chat — the agent logs in for you. Credentials are parameterized into `process.env` references and never written into the spec. See [Your first session](/docs/get-started/first-session).

</Callout>

## Save the session

Click **Save as Spec** on the Result card. The verified flow becomes a `__vibe_tests__/<slug>.spec.ts` file — plain `@playwright/test` code that runs in your CI with no Hover dependency.

That's the loop: speak / type → agent verifies → crystallize once → CI replays forever.

## Run the spec in CI

The saved spec is plain Playwright. Run it anywhere with no AI, no tokens, no key:

```bash
npx playwright test __vibe_tests__
```

Point it at any environment with `BASE_URL` (and the `HOVER_<LABEL>_*` account secrets); the same spec runs against local, staging, or a PR preview. Hover can generate a GitHub Actions workflow that runs them on every PR.

## Other surfaces

- **`hover run "<prompt>"`** — author a spec from the terminal, no editor needed. A living secondary surface; see [CLI](/docs/reference/cli).
- **Bundler plugins (frozen)** — the original in-page widget (`vite-plugin-hover`, `@hover-dev/astro` / `nuxt` / `next`, `webpack-plugin-hover`, wired by `npx @hover-dev/cli setup`). Still works, still documented, but no longer the recommended path — see [Install → Frozen bundler-plugin path](/docs/get-started/install#frozen-bundler-plugin-path).
