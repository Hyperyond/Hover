# Features

What Hover does today, in the VS Code extension and its living surfaces.

## Modes

- **Flow** — describe a flow; the agent drives your app and crystallizes the verified run into a `.spec.ts` for CI.
- **🟢 QA Testing** — autonomously explores the whole app to find defects → a findings report (with a coverage map) and ✨ promotable specs, bounded by an intensity budget (Quick / Standard / Deep). Two **[API & security](/docs/features/api-test/)** capability toggles ride on top: **API testing** routes the debug Chrome through a local HTTPS MITM and replays captured calls with mutations, probing IDOR / broken authorization / parameter tampering (→ `.api-test.spec.ts` CI gates); **penetration testing** runs offensive checks (SQLi / XSS / SSTI / SSRF / open-redirect / IDOR) against your **own** dev app, as a separate destructive pass (off by default) → a report.

## Authoring

- **[@account multi-environment credentials](/docs/get-started/first-session/#step-2-set-up-an-environment-optional)** — define test accounts per environment, mention `@account` in chat, and the agent logs in. Credentials are parameterized into `process.env` references — never written into the spec, the JSDoc, or the sidecar — and the same names export to your CI secrets in one click.
- **white-box `codeContext`** — an opt-in, read-only source reader the agent can use to ground its testing in your code. Off by default; fenced to read-only.

## Output

- **[Save as Spec](/docs/features/save-as-spec/)** — `__vibe_tests__/<slug>.spec.ts` using `getByRole / getByLabel / getByTestId`. Runs in CI without Hover. Every interaction is wrapped in a visibility prelude so UI drift fails in ~3 s with `Locator expected to be visible` instead of a 30 s actionability timeout.
- **[Save as an API-test spec](/docs/features/api-test-spec/)** — `__vibe_tests__/<slug>.api-test.spec.ts` using Playwright's `request` fixture. Crystallises authz / IDOR / parameter-tampering probes into CI-runnable regression specs.
- **[Structured spec output](/docs/features/structured-output/)** — the saved spec grows an architecture: `test.step` stages, popup / new-tab `Promise.all` pairing, a `.hover/<slug>.json` sidecar, and Page Objects + fixtures lifted from flows repeated across specs.
- **[Optimize a spec](/docs/features/optimization-pass/)** — an optional, off-by-default AI pass polishes a deterministic draft into a diff-reviewed candidate: adds the assertions the run observed, flags buggy behaviour `// KNOWN BUG`, always keeps the original. Run it from the Dashboard's ✨ Optimize.
