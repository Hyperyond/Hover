# Features

What Hover does today, in the VS Code extension and its living surfaces.

## Modes

- **[Security testing](/docs/features/security/)** — 🟠 **Security** mode routes the debug Chrome through a local HTTPS MITM; the agent lists / inspects / replays captured API calls with mutations, probing for IDOR / broken authorization / parameter tampering. 🔴 **Pentest** mode runs offensive checks (SQLi / XSS / SSTI / SSRF / open-redirect / IDOR) against your **own** dev app. Findings crystallize into `.security.spec.ts` CI gates or a report.

## Authoring

- **[@account multi-environment credentials](/docs/get-started/first-session/#step-2-set-up-an-environment-optional)** — define test accounts per environment, mention `@account` in chat, and the agent logs in. Credentials are parameterized into `process.env` references — never written into the spec, the JSDoc, or the sidecar — and the same names export to your CI secrets in one click.
- **white-box `codeContext`** — an opt-in, read-only source reader the agent can use to ground its testing in your code. Off by default; fenced to read-only.

## Output

- **[Save as Spec](/docs/features/save-as-spec/)** — `__vibe_tests__/<slug>.spec.ts` using `getByRole / getByLabel / getByTestId`. Runs in CI without Hover. Every interaction is wrapped in a visibility prelude so UI drift fails in ~3 s with `Locator expected to be visible` instead of a 30 s actionability timeout.
- **[Save as Security spec](/docs/features/security-spec/)** — `__vibe_tests__/<slug>.security.spec.ts` using Playwright's `request` fixture. Crystallises authz / IDOR / parameter-tampering probes into CI-runnable regression specs.
- **[Findings cards](/docs/features/findings-cards/)** — bugs and observations the agent flagged, severity-coloured. First-class output, not buried in narration.
- **[Structured spec output](/docs/features/structured-output/)** — the saved spec grows an architecture: `test.step` stages, popup / new-tab `Promise.all` pairing, a `.hover/<slug>.json` sidecar, and Page Objects + fixtures lifted from flows repeated across specs.
- **[Optimize a spec](/docs/features/optimization-pass/)** — an optional, off-by-default AI pass polishes a deterministic draft into a diff-reviewed candidate: adds the assertions the run observed, flags buggy behaviour `// KNOWN BUG`, always keeps the original. Run it from the Specs view's ✨ Optimize.
