# Agent registry

Hover bundles no AI runtime (BYO-CLI). In the **MCP-first** flow, the intelligence is *your* agent — you add `@hover-dev/mcp` to whatever MCP-capable coding agent you already run (Claude Code, Cursor, …), and it talks to its own provider; Hover holds no key.

Hover's **engine** (`@hover-dev/core`) also knows how to spawn a coding-agent CLI directly — this is what the optional VS Code cockpit's chat uses. Each supported CLI is a single `AgentDescriptor` entry in [`packages/core/src/agents/registry.ts`](https://github.com/Hyperyond/Hover/blob/main/packages/core/src/agents/registry.ts); the engine auto-detects the first one installed on your `PATH`.

## Supported agents

| Agent | id | Sandbox | Notes |
|---|---|---|---|
| Claude Code | `claude` | hard | `--strict-mcp-config` + `--allowedTools mcp__playwright` + a per-tool `--disallowedTools` deny list. Honours `--max-budget-usd`. stream-json on stdout. The recommended primary. |
| OpenAI Codex | `codex` | soft | `--sandbox read-only` + a strict `developer_instructions` system prompt (Codex has no CLI-level built-in-tool deny list). JSONL on stdout. ⚠ in the dropdown. |
| Google Gemini | `gemini` | soft | Gemini CLI. `--output-format stream-json`. MCP via `~/.gemini/settings.json` (not a per-invocation `--mcp-config`). No `--append-system-prompt`, so the Hover preface prepends to the user prompt. ⚠ in the dropdown. |
| Local LLM (Qwen Code) | `qwen` | soft | Qwen Code CLI, surfaced as **Local LLM** — a model-agnostic OpenAI-compatible host. Point it at a self-hosted endpoint (Ollama / LM Studio / vLLM) by setting a Base URL in Settings; Hover runs it with `OPENAI_BASE_URL`. Real `--append-system-prompt`; MCP via `~/.qwen/settings.json`. ⚠ in the dropdown. |

`cursor-agent` and `aider` were prototyped earlier and removed; the registry today is the four agents above.

## Local models

The `qwen` agent doubles as Hover's **Local LLM** path. Set the **Base URL** (`hover.localBaseUrl`) and **model** (`hover.localModel`) in Settings and Hover spawns Qwen Code with `OPENAI_BASE_URL` pointed at your endpoint — and a placeholder `OPENAI_API_KEY` if you haven't set one, so no cloud key is required. See [Use an API key](/docs/get-started/api-key/).

## The `AgentDescriptor` interface

```ts
interface AgentDescriptor {
  id: string;                                   // e.g. 'claude', 'codex', 'gemini', 'qwen'
  binName: string;                              // executable name on PATH
  protocol: 'argv' | 'stdin' | 'acp' | 'pi-rpc';
  streamFormat: 'stream-json' | 'sse' | 'plain-text' | 'json-lines';
  sandboxStrength: 'hard' | 'soft';
  display: AgentDisplay;                        // label, tagline, homepage, installHint
  buildArgs(opts: InvokeOptions): string[];
  parseEvent(line: string, state?: ParserState): InvokeEvent[];
  onStreamEnd?(exitCode: number | null, state?: ParserState): InvokeEvent | null;
}
```

## Sandbox strength

- **`hard`** — the agent CLI accepts a deny / allow list that effectively removes built-in tools (shell, file edit, …) so the only callable surface is whatever MCP servers Hover configures. Example: Claude Code's `--strict-mcp-config --allowedTools mcp__playwright --disallowedTools "Bash Edit Write Read …"`.
- **`soft`** — no equivalent flag exists. Hover constrains side-effects via OS-level sandbox flags (e.g. Codex's `--sandbox read-only`) and leans on a strict `developer_instructions` system prompt to nudge the agent toward MCP-only behavior. A determined or hallucinating agent *could* still try a built-in shell call.

Hover marks soft-sandbox agents with a ⚠ badge in the agent dropdown.

## Adding a new agent

Write a descriptor under `packages/core/src/agents/`, import it in `registry.ts`, and append it to the `AGENTS` constant. Pick the closest existing peer as a template:

- **Hard-sandbox CLI** (has an allow/deny tool list) — copy `claude.ts`.
- **Soft-sandbox CLI** with a system-prompt directive — copy `codex.ts`, `gemini.ts`, or `qwen.ts`.

The detect / argv / invoke / service / chat chain picks it up automatically — no other changes. PRs welcome.
