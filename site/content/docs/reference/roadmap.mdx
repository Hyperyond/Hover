# Roadmap

What's shipped, what's in flight.

## Today ‚Äî MCP-first, four surfaces

Hover is an open-source Vibe Testing suite organised around four surfaces and one artifact:

| Surface | Role | Status |
|---|---|---|
| **MCP** ‚Äî `@hover-dev/mcp` | **author** ‚Äî add it to your own agent; `/mcp__hover__test_app` explores + crystallizes a suite | ‚úÖ Shipped (on npm) |
| **VS Code** ‚Äî `hover-dev` | **review** ‚Äî an optional Business Map + Dashboard cockpit; drives no agent | ‚úÖ Shipped (Marketplace) |
| **CI** | **run** ‚Äî the crystallized specs run on every PR as plain Playwright | ‚úÖ Shipped |
| **Cloud** | **watch** ‚Äî hosted parallel runs, monitoring, flakiness dashboard, on-failure self-heal | Planned |

The through-line is the artifact: owned, portable `@playwright/test` in your repo and your CI. The AI authors it once; nothing AI runs after.

## Recently shipped

- **[API testing](/docs/features/api-testing/).** Hover passively captures the app's browser traffic off the same CDP connection (no proxy) and crystallizes the checks worth pinning into a plain `*.api-test.spec.ts` ‚Äî verified with `replay_request` first, so no status code is confabulated.
- **[Self-heal ‚Äî mode A (local, on-demand)](/docs/features/self-heal/).** `/mcp__hover__heal` replays a saved spec against the live app, finds the first drifted step, and re-grounds only that step. Record == replay preserved.
- **[Page Object extraction](/docs/features/page-objects/).** NON-login flows repeated across specs are lifted into `pages/` + `fixtures.ts` and the specs folded onto them ‚Äî deterministically.
- **[Auth as a fixture](/docs/features/auth/).** A detected login is lifted into an `auth.setup.ts` setup project that logs in once and reuses `storageState`; the password is redacted to `process.env.HOVER_PASSWORD`.

## What's next

- **A deeper test-knowledge graph.** A richer `.hover/` memory layer ‚Äî a relationship graph linking flows, pages, and the endpoints they touch, plus coverage-gap detection ‚Äî over the Business Map + remembered rules that ship today. No source uploaded.
- **CI-driven self-heal (mode B).** A red CI run offering to heal the drifted spec automatically. Local, on-demand heal ships today; the CI-triggered path is planned.
- **Hover Cloud (planned, not shipped).** A hosted layer over the specs you already own: parallel runs, scheduled monitoring, and a flakiness dashboard. Authoring always stays local and free; the cloud only ever *runs and monitors* specs you already own ‚Äî never authoring lock-in.
