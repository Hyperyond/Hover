## What's next

- **Chrome extension** as a possible sibling product. Targets the "AI tests any live site" use case (staging URLs, third-party sites, multi-origin flows). Likely a separate `hover-extension` repo because Chrome Web Store releases are manual and would mis-fit this monorepo's auto-publish workflow.
- **Hover Cloud (planned, not shipped).** A hosted layer over your local specs — parallel runs, scheduled monitoring, a flakiness dashboard, and on-failure AI self-heal of UI-drifted specs (replaying a spec's recorded intent against the current UI and proposing a diff — batched across the specs Playwright reports as failing). Authoring always stays local and free; the cloud only ever *runs and monitors* specs you already own.

## Beyond v0.15.x

- **Voice mode Pro** (opt-in) — Deepgram Nova-3 streaming STT + ElevenLabs Flash / Cartesia Sonic TTS as an env-var-gated upgrade for users who want sub-100ms TTFA or higher-fidelity 中文 recognition. LLM stays on the user's local CLI agent — only the I/O adapters are cloud. Brings new privacy boundary (audio leaves the browser); will be opt-in, never default.
- **AI-compiled spec output** — ✅ **shipped** as the off-by-default [optimization pass](/docs/features/optimization-pass/): the deterministic translator always emits the spec, and an optional LLM pass polishes it into a diff-reviewed candidate (original always kept). The AI is an authoring-time aid, never a CI dependency.

For real-time tracking, see [github.com/Hyperyond/Hover/issues](https://github.com/Hyperyond/Hover/issues).
